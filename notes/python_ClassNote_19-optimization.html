Review of Optimization Modeling¶

Optimization is the process of finding the best solution to a problem, given a limited set of options. Solving an optimization typically involves three distinct steps:

  • Formulating the optimization problem - Define decision variables and specify objective function and constraints, expressed as a function of the decision variables, fixed input parameters, and any intermediate calculations

  • Solving the optimization problem - Identify and employ an appropriate algorithm to identify the best set of decision variables, given the objective function and the constraints

  • Performing sensitivity analysis - Explore relevant alternative scenarios and understand the impact on the optimal decision variables and objective function

Island Sunglasses¶

Recap: Goal is to determine optimal price for new product.

Demand: $Q = M(1 - \frac{P}{U})$

where:

  • Q: Quantity demanded
  • P: Price
  • M: Maximum market size (Q when P = 0) = 10,000
  • U: Maximum price (generates Q = 0) = \$50

Cost = F + VQ

where:

  • F: Fixed production cost = \$50,000
  • V: Variable production cost = \$10/unit

Formulation

Fixed input parameters: Maximum market size (M), maximum price (U), fixed production cost (F), variable production cost (V)

Decision variable: Price (P)

Intermediate calculations:

  • Demand: $Q = M(1 - \frac{P}{U})$
  • Revenue: $PQ$
  • Cost: $F + VQ$

Objective function: $max$ Profit = Revenue - Cost

Constraints:

  • Maximum price: $P \le U$
  • Non-negativity: $P \ge 0$

Function Minimization Approach¶

The scipy.optimize package offers an entire library of optimization solvers, including the fminbound function that can be used to solve a minimization problem involving a single decision variable. The function accepts the following (primary) arguments:

  • fun - A function object that calculates the (scalar) objective function value. The function should accept the decision variable (x) as its first argument, but it may accept other arguments as well (see args below). Note that for maximization problems, the function should return the negated objective function value (i.e., a maximization objective is equivalent to minimizinig the negative objective function).

  • x1, x2 - Floats that defines the search bounds (x1 - min, x2 - max) for the decision variable.

  • args - A sequence of additional arguments to pass to the objective function object.

The function returns the optimal decision variable value (xopt).

For additional details, please see https://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.fminbound.html#scipy.optimize.fminbound.

In [11]:
# Define objective function
def profit(x, M=10000, U=50, F=50000, V=10):
    # Perform intermediate calculations
    Q = M * (1 - x / U)
    revenue = x * Q
    cost = F + V * Q
    
    # Return profit
    return -(revenue - cost)

# Solve optimization problem
Popt = so.fminbound(profit, x1=0, x2=U, args=())

Giant Motor Company¶

The Giant Motor Company has just opened two new manufacturing plants to produce its Lyra, Libra, and Hydra models, which have respective total demands of 1.4, 1.1, and 0.8 million units.

Plant 1 has a capacity to produce up to 1.5 million cars (of any type), while Plant 2 can produce up to 1.8 million cars (of any type).

Due to variation in cost efficiencies at the respective plants, the profit margins per unit sold (in thousands) depend on the plant where the vehicle was produced, noting that Plant 1 does not have the ability to produce the Hydra model:

Formulation

Fixed input parameters:

  • Demands of 1.4, 1.1, and 0.8 million units for the Lyra, Libra, and Hydra models, respectively
  • Plant capacities of 1.5 and 1.8 million units for Plant 1 and Plant 2, respectively
  • Plant-based per-unit profit margines, reported above

Decision variables: Let $Lyra_1$, $Lyra_2$, $Libra_1$, $Libra_2$, and $Hydra_2$ serve as decision variables that represent the number of respective models to be produced (in millions) at Plant 1 and Plant 2, respectively

Objective Function:

$max$ Profit (in billions) = $2.5 * Lyra_1 + 2.3 * Lyra_2 + 3 * Libra_1 + 3.5 * Libra_2 + 4.8 * Hydra_2$

Constraints:

Demand Limits (excess production has no benefit with respect to objective function)

  1. $Lyra_1 + Lyra_2 \le 1.4$
  2. $Libra_1 + Libra_2 \le 1.1$
  3. $Hydra_2 \le 0.8$

Supply Limits

  1. $Lyra_1 + Libra_1 \le 1.5$
  2. $Lyra_2 + Libra_2 + Hydra_2 \le 1.8$

Non-negativity

  1. $Lyra_1, Lyra_2, Libra_1, Libra_2, Hydra_2 \ge 0$

Solution Procedure¶

In this case, given that we have 5 decision variables that can take on a broad range of values (e.g., from 0 up to the maximum demand for each model type), a brute-force approach would not be efficient. Therefore, we need an algorithm that supports solving the problem more directly.

The scipy.optimize package offers the linprog function that can be used to solve linear optimization problems with one or more decision variables. The linprog function requires that the formulation be expressed in linear algebraic form:

$min \quad c^T * x$

subject to:

$A_{ub} * x \le b_{ub}$

$A_{eq} * x = b_{eq}$

Therefore, the arguments to the linprog function are:

  • c - A 1-dimensional array of objective function coefficients

  • $A_{ub}$ - A 2-dimensional array of decision variable coefficients for each upper-bound constraint (i.e., $\le$)

  • $b_{ub}$ - A 1-dimensional array of right-hand side constants for each upper-bound constraint

  • $A_{eq}$ - A 2-dimensional array of decision variable coefficients for each equality constraint (i.e., $=$)

  • $b_{eq}$ - A 1-dimensional array of right-hand side constants for each equality constraint

  • bounds - An (optional) sequence of (lower, upper) bounds for each decision variable. Note that these bounds can also be enforced via the upper-bound constraints (e.g., $-x \le 0$ and $x \le 10$).

The function returns an OptimizeResults object, for which we can extract the optimal solution and optimal objective function value via the .x and .fun attributes, respectively.

In conclusion, it would not help The Giant Motor Company to increase the production capacity of Plant 1, as the production schedule and optimal profit would remain the same. However, if The Giant Motor Company were to increase the production capacity of Plant 2 by 100,000 units (i.e., to 1.9M units), it could fulfill all of the Libra demand from Plant 2, where producing the model is more profitable. The additional 100,000 units produced at Plant 2 would earn an additional \$500 of profit per unit = \\$50M, leading to total profits of \$11.19B. Therefore, the shadow price of an additional unit of production capacity at Plant 2 is equal to \\$500. In other words, The Giant Motor Company would be willing to pay up to \$500 for each additional unit of production capacity at Plant 2, for up to 100,000 additional units. Beyond that point, there would be no additional benefit.
